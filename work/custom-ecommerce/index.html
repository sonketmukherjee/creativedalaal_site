The brief

One product, many prices.

Off-the-shelf e-commerce assumes one price per product. This client’s reality was the opposite: prices changed by user type, by quantity, and with setup and delivery fees layered on top.

On top of that, customers needed to design the product themselves — and the price had to react to those choices.

The unique build

A custom pricing engine, with a Zakeke customiser wired into it.

I built bespoke pricing logic that calculates by user type and quantity, then adds setup and delivery fees — surfaced through pricing tables that change per user type so each buyer sees only what’s relevant to them.

Then I integrated the Zakeke product design customiser directly into that custom pricing, so as a customer personalises their product, the price stays accurate in real time. Standard plugins can’t do this — it had to be built.
